## Introduction

[SoybeanAdmin](https://docs.soybeanjs.cn/zh/) is an elegant, high-quality, and powerful backend management template based on the latest front-end technology stack, including Vue3, Vite5, TypeScript, Pinia, and UnoCSS. It has a rigorous code structure and an automated file routing system, and it comes with rich theme configurations and components. `SoybeanAdmin` provides a one-stop backend management solution that is ready to use without additional configuration. It is also an excellent practice for quickly learning cutting-edge technologies.

[Vue3NaiveAdmin](https://github.com/mrzym99/vue3-naive-admin) follows the [SoybeanAdmin development standards](https://docs.soybeanjs.cn/zh/standard/) and retains all the features of [SoybeanAdmin](https://docs.soybeanjs.cn/zh/). It also adds some advanced components (please refer to the README inside the components for usage). For the usage of the built-in functions of the framework, please refer to the [SoybeanAdmin official documentation](https://docs.soybeanjs.cn/zh/guide/intro.html).

The backend of this project is supported by [Nest-Admin](https://github.com/mrzym99/nest-admin): [https://github.com/mrzym99/nest-admin](https://github.com/mrzym99/nest-admin).

## Features

- **Cutting-edge technology stack**: Utilizes the latest popular technologies such as Vue3, Vite5, TypeScript, Pinia, and UnoCSS.
- **Clear project architecture**: Uses pnpm monorepo architecture, which is clear, elegant, and easy to understand.
- **Strict code standards**: Follows the [SoybeanJS standards](https://docs.soybeanjs.cn/zh/standard), integrates eslint, prettier, and simple-git-hooks to ensure code standards.
- **TypeScript support**: Provides strict type checking to enhance code maintainability.
- **Rich theme configurations**: Comes with a variety of theme configurations that perfectly integrate with UnoCSS.
- **Built-in internationalization**: Easily supports multiple languages.
- **Automated file routing system**: Automatically generates route imports, declarations, and types. For more details, please refer to [Elegant Router](https://github.com/soybeanjs/elegant-router).
- **Flexible permission routing**: Supports both front-end static routes and back-end dynamic routes.
- **Rich page components**: Includes a variety of pages and components, such as 403, 404, 500 pages, layout components, tag components, and theme configuration components.
- **Command-line tools**: Built-in efficient command-line tools for git commits, file deletion, and publishing.
- **Mobile adaptation**: Perfectly supports mobile devices with responsive layout.
- **Advanced components**: Built-in table, form, and detail components for rapid page development (written by Xiao Zhang, inexperienced, please be gentle with criticism).

## Usage

**Environment Preparation**

Ensure that your environment meets the following requirements:

- **git**: You need git to clone and manage project versions.
- **NodeJS**: >=18.12.0, recommended 18.19.0 or higher.
- **pnpm**: >= 8.7.0, recommended 8.14.0 or higher.

**Clone the Project**

```bash
git clone https://github.com/mrzym99/vue3-naive-admin
```

**Install Dependencies**

```bash
pnpm i
```
> Since this project uses pnpm monorepo management, please do not use npm or yarn to install dependencies.

**Start the Project**

```bash
pnpm dev
```

**Build the Project**

```bash
pnpm build
```

**Code Synchronization**

1. Add the Vue3NaiveAdmin git address to your repository:
```bash
git remote add otherOrigin https://github.com/mrzym99/vue3-naive-admin.git
```
2. Fetch the code:
```bash
git fetch otherOrigin
```
3. Use `cherry-pick` to select the git commits you need to update:
```bash
git cherry-pick [commit id]
```
4. If there are conflicts, resolve them first, then execute the following command and save with vim:
```bash
git cherry-pick --continue
```

## Docker Build

This project uses Docker to build and run the project. You can follow the steps below to build and run the project:

**Environment Requirements**

Your environment must meet the following requirements:
- Stable versions of Docker and Docker Compose.

**Notes**

- `docker-compose.yml` corresponds to the development environment (dev), while `docker-compose.prod.yml` corresponds to the production environment (prod).

1. Build the project:
```bash
pnpm run docker:build:dev # For development environment
pnpm run docker:build:prod # For production environment
```

2. Run the project:
```bash
pnpm run docker:up:dev # For development environment
pnpm run docker:up:prod # For production environment
```

3. Stop and remove:
```bash
pnpm run docker:down # Stop and remove
pnpm run docker:down:v # Stop and remove containers, and remove data volumes (use with caution)
```
[Docker Compose Command List](https://www.cnblogs.com/xyh9039/p/18540766)

If you encounter build failures, it may be due to issues with your Docker environment. You can switch to a different Docker source address. Here, we recommend using [Alibaba Cloud Mirror Acceleration](https://cr.console.aliyun.com/cn-hangzhou/instances/mirrors).
